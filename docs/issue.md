## 🔹 [Auth] 구글 소셜 로그인 이슈/해결 정리 (2026-02-20)

구글 소셜 로그인은 초기에는 로그인 자체는 성공해도 프로필 닉네임/이미지가 비어 보이거나, 로그아웃 후 재로그인 시 이전 구글 계정으로 자동 진입되는 문제가 있었다. 1차 원인은 OAuth 리다이렉트 직후 `user_metadata` 반영 타이밍과 DB 프로필 동기화 기준이 불안정했던 점이며, 이를 보완하기 위해 로그인 시 세션을 재확인하고 프로필 생성/조회 기준을 명확히 분리했다. 이후 `user_profiles`에 `address(메일)` 컬럼을 추가하고, address 기반으로 신규/기존 유저를 판별하도록 로직을 단순화했다(신규: 프로필 생성, 기존: DB 조회). 2차 원인은 구글 계정 선택창 미노출로 인한 계정 전환 불가였고, OAuth `prompt=select_account`를 적용해 매 로그인 시 계정 선택이 가능하도록 수정했다. 추가로 계정 삭제 플로우에서는 JWT/권한 검증 문제와 웹 확인 다이얼로그 흐름 이슈를 정리해, 사용자 토큰 기반 호출과 삭제 후 상태 초기화까지 일관되게 동작하도록 안정화했다.

## 🔹 [Account] 계정 삭제 이슈/해결 정리 (2026-02-20)

계정 삭제 기능은 초기에는 버튼 클릭 이후 반응이 없거나, 삭제 후 재가입 시 홈/북마크 무한 로딩, 설정 저장 불가가 함께 발생했다. 1차 원인은 웹 환경에서 `Alert.alert` 동작이 불안정해 확인 다이얼로그 흐름이 끊긴 점이었고, 이를 `window.confirm` 분기로 보완해 삭제 플로우가 정상 실행되도록 수정했다. 2차로는 Edge Function 호출 시 CORS preflight(OPTIONS) 단계에서 차단되어 `FunctionsFetchError`가 발생했으며, 함수 응답 헤더에 `Access-Control-Allow-Methods`, `Access-Control-Max-Age`를 포함해 브라우저 호출 호환성을 강화했다. 3차로는 `PGRST116 (2 rows)`로 프로필/설정 보장 로직이 실패했는데, 원인은 단일 객체를 기대한 쿼리에서 중복 데이터가 반환된 케이스였다. `limit(1)` 기반 조회로 방어하고, `user_settings` 중복 정리 및 `user_id` UNIQUE 인덱스 마이그레이션을 추가했다. 또한 신규 유저 생성 시 `ensureNewUserData`를 통해 `user_settings` 기본값과 `bookmark_folders` 기본 폴더를 즉시 생성하도록 통합했으며, 북마크 화면은 `useAuth` 기반 재로딩/에러 처리로 무한 로딩을 방지했다.
